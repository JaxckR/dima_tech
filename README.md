# DimaTech — тестовое задание

REST API приложение для управления пользователями, счетами и платежами.

## Стек

- **Python 3.14**
- **FastAPI** — веб-фреймворк
- **SQLAlchemy 2.0** (async) — ORM
- **PostgreSQL** — база данных
- **Alembic** — миграции
- **Dishka** — DI
- **Docker Compose** — развёртывание

## Тестовые учётные данные

После применения миграций в базе будут созданы следующие тестовые записи:

| Роль          | Email               | Пароль       |
|---------------|---------------------|--------------|
| Пользователь  | `user@example.com`  | `user12345`  |
| Администратор | `admin@example.com` | `admin12345` |

У тестового пользователя(не администратора) также создаётся счёт с балансом `0`.

## Вариант 1: запуск через Docker Compose (рекомендуется)

### Требования

- Docker
- Docker Compose

### Шаги

1. Скопируйте файл с переменными окружения:

   ```bash
   cp .env.dist .env
   ```

2. Отредактируйте `.env`

3. Запустите проект:

   ```bash
   docker compose up --build
   ```

   При старте контейнера приложения автоматически применяются миграции (включая seed-данные с тестовыми пользователями).

4. Приложение будет доступно по адресу:

   ```
   http://localhost:8000
   ```

   Документация Swagger UI:

   ```
   http://localhost:8000/docs
   ```

## Вариант 2: запуск без Docker (локально)

### Требования

- Python 3.14+
- [uv](https://docs.astral.sh/uv/) (менеджер пакетов)
- Запущенный локально PostgreSQL (создайте базу данных заранее)

### Шаги

1. Установите зависимости:

   ```bash
   uv sync
   ```

2. Скопируйте файл с переменными окружения и укажите свои значения для подключения к локальному PostgreSQL:

   ```bash
   cp .env.dist .env
   ```


3. Экспортируйте переменные окружения из `.env`:

   ```bash
   export $(cat .env | xargs)
   ```

4. Примените миграции:

   ```bash
   uv run alembic upgrade head
   ```

5. Запустите приложение:

   ```bash
   uv run python -m dima_tech.bootstrap.entrypoints.web
   ```

6. Приложение будет доступно по адресу:

   ```
   http://localhost:8000
   ```

## Структура API

### Авторизация

- `POST /auth/login` — авторизация по email/password (пользователь и администратор)
- `GET /auth/me` — получить данные о себе (id, email, full_name)

### Пользователь

- `GET /accounts/me` — список своих счетов с балансами
- `GET /payments/me` — список своих платежей

### Администратор

- `POST /users` — создать пользователя
- `PATCH /users/{id}` — обновить пользователя
- `DELETE /users/{id}` — удалить пользователя
- `GET /users` — список пользователей со счетами и балансами

### Вебхук платёжной системы

- `POST /webhooks/payment` — обработка вебхука пополнения баланса

  Пример тела запроса:

  ```json
  {
    "transaction_id": "5eae174f-7cd0-472c-bd36-35660f00132b",
    "user_id": 1,
    "account_id": 1,
    "amount": 100,
    "signature": "7b47e41efe564a062029da3367bde8844bea0fb049f894687cee5d57f2858bc8"
  }
  ```

  Подпись формируется как SHA256-хеш строки конкатенации значений в порядке `account_id`, `amount`, `transaction_id`, `user_id` и секретного ключа из конфигурации (`secret_key`).
